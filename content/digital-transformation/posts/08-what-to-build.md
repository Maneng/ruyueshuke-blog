## 引言：不是所有系统都要自研

上一篇我们讲了自研vs采购的决策框架。但具体到跨境电商，哪些系统适合自研？

**跨境电商有其特殊性**：
- 多渠道、多币种、多仓库
- 复杂的订单逻辑
- 特殊的仓储需求
- 跨境合规要求

这些特殊性决定了，有些系统必须自研，有些系统采购更划算。

---

## 一、跨境电商的业务特殊性

### 1.1 多渠道复杂性

**渠道类型**：
- 平台渠道：Amazon、eBay、Wish、速卖通
- 独立站：Shopify、自建站
- 社交电商：TikTok Shop、Facebook Shop

**每个渠道的差异**：

| 渠道 | 订单结构 | API方式 | 结算周期 | 特殊规则 |
|-----|---------|--------|---------|---------|
| Amazon | 复杂 | REST API | 14天 | FBA/FBM |
| eBay | 中等 | REST API | 即时 | 拍卖模式 |
| Shopify | 简单 | GraphQL | 即时 | Webhook |
| 速卖通 | 复杂 | REST API | 15天 | 纠纷规则 |

**挑战**：
- 每个渠道的订单格式不同
- 每个渠道的API不同
- 每个渠道的规则不同
- 需要统一管理

### 1.2 多币种复杂性

**涉及的币种**：
- 销售币种：USD、EUR、GBP、JPY、AUD...
- 采购币种：CNY、USD
- 结算币种：USD、CNY

**挑战**：
- 汇率波动影响利润
- 多币种成本核算
- 汇兑损益处理

### 1.3 多仓库复杂性

**仓库类型**：
- 国内仓：深圳、义乌
- 海外仓：美国、欧洲、日本
- FBA仓：亚马逊仓库
- 第三方仓：海外第三方仓

**挑战**：
- 库存分布在多个仓库
- 不同仓库的操作流程不同
- 跨仓调拨
- 库存同步

### 1.4 订单逻辑复杂性

**复杂场景**：
- 多仓拆单：一个订单从多个仓库发货
- 合单发货：多个订单合并发货
- 预售订单：先卖后采
- 组合商品：一个SKU包含多个实物
- 赠品逻辑：满赠、买赠

**挑战**：
- 标准系统很难满足这些复杂逻辑
- 每个企业的规则还不一样

## 二、必须自研的系统

### 2.1 OMS订单系统

**为什么必须自研？**

| 原因 | 说明 |
|-----|-----|
| 多渠道接入 | 每个渠道API不同，需要定制适配器 |
| 订单逻辑复杂 | 拆单、合单、预售等逻辑高度定制 |
| 核心竞争力 | 订单处理效率直接影响客户体验 |
| 快速迭代 | 业务变化快，需要快速响应 |

**采购的问题**：
- 标准OMS无法满足多渠道需求
- 定制化成本高，响应慢
- 被厂商绑架

**自研的核心模块**：
```
┌─────────────────────────────────────────────────────┐
│                    OMS自研范围                       │
├─────────────────────────────────────────────────────┤
│  ✓ 多渠道订单接入（适配器）                          │
│  ✓ 订单处理引擎（审核、拆分、合并）                  │
│  ✓ 库存预占与管理                                   │
│  ✓ 履约路由（选仓、选物流）                         │
│  ✓ 订单状态管理                                     │
│  ✓ 异常订单处理                                     │
└─────────────────────────────────────────────────────┘
```

### 2.2 WMS仓储系统（核心模块）

**为什么建议自研？**

| 原因 | 说明 |
|-----|-----|
| 仓储策略定制 | 上架策略、拣货策略需要根据业务定制 |
| 与OMS深度集成 | 库存同步、出库指令需要紧密配合 |
| 效率是核心 | 仓储效率直接影响成本和时效 |

**可以采购的情况**：
- 使用第三方仓库，仓库提供WMS
- 业务简单，标准WMS能满足

**自研的核心模块**：
```
┌─────────────────────────────────────────────────────┐
│                    WMS自研范围                       │
├─────────────────────────────────────────────────────┤
│  ✓ 库位管理                                         │
│  ✓ 入库管理（收货、质检、上架）                      │
│  ✓ 出库管理（波次、拣货、复核）                      │
│  ✓ 库存管理（盘点、调拨）                           │
│  ✓ 策略引擎（上架策略、拣货策略）                    │
│  ○ 设备对接（可用第三方）                           │
└─────────────────────────────────────────────────────┘
```

### 2.3 库存中台

**为什么建议自研？**

| 原因 | 说明 |
|-----|-----|
| 多仓库存汇总 | 需要汇总多个仓库的库存 |
| 可售库存计算 | 复杂的可售库存逻辑 |
| 与OMS/WMS集成 | 需要实时同步 |

**核心功能**：
- 多仓库存汇总
- 可售库存计算（考虑预占、锁定、在途）
- 库存预警
- 库存分析

## 三、建议采购的系统

### 3.1 财务系统

**为什么建议采购？**

| 原因 | 说明 |
|-----|-----|
| 合规要求高 | 财务系统需要符合会计准则 |
| 标准化程度高 | 财务流程相对标准 |
| 成熟产品多 | 金蝶、用友等产品成熟 |
| 自研风险大 | 财务出错影响严重 |

**推荐产品**：

| 产品 | 适用规模 | 价格 | 特点 |
|-----|---------|-----|-----|
| 金蝶云星辰 | 中小企业 | 5-20万/年 | 云端、易用 |
| 金蝶云星空 | 中大企业 | 20-50万/年 | 功能全 |
| 用友U8+ | 中小企业 | 10-30万 | 本地部署 |
| 用友NC | 大企业 | 50万+ | 集团管控 |

**集成方案**：
- OMS/WMS的业务数据通过接口同步到财务系统
- 财务系统负责凭证生成、报表输出

### 3.2 HR系统

**为什么建议采购？**

| 原因 | 说明 |
|-----|-----|
| 非核心业务 | HR不是跨境电商的核心竞争力 |
| SaaS成熟 | 钉钉、企业微信、北森等 |
| 成本低 | SaaS按人头收费，成本可控 |

**推荐产品**：

| 产品 | 价格 | 特点 |
|-----|-----|-----|
| 钉钉 | 免费/低价 | 考勤、审批、沟通 |
| 企业微信 | 免费/低价 | 与微信打通 |
| 北森 | 按人头 | 专业HR系统 |

### 3.3 TMS物流系统

**为什么建议采购/SaaS？**

| 原因 | 说明 |
|-----|-----|
| 物流商对接复杂 | 每个物流商API不同 |
| SaaS产品成熟 | 快递100、菜鸟等 |
| 非核心竞争力 | 物流执行不是核心 |

**推荐方案**：
- 使用物流聚合平台（如快递100企业版）
- 或使用物流商提供的系统

**自研的情况**：
- 物流是核心竞争力
- 有特殊的物流需求

### 3.4 CRM客户系统

**为什么建议采购？**

| 原因 | 说明 |
|-----|-----|
| 标准化程度高 | CRM功能相对标准 |
| SaaS产品多 | Salesforce、纷享销客等 |
| 非核心业务 | 跨境电商以平台为主，CRM需求弱 |

**注意**：
- 如果做独立站，CRM可能更重要
- 可以考虑自研或深度定制

## 四、可选自研的系统

### 4.1 ERP采购模块

**自研的情况**：
- 采购流程复杂
- 与OMS/WMS深度集成需求
- 有技术团队能力

**采购的情况**：
- 采购流程标准
- 可以用金蝶/用友的采购模块

### 4.2 BI报表系统

**自研的情况**：
- 报表需求高度定制
- 数据分析是核心竞争力

**采购的情况**：
- 使用标准BI工具（如帆软、Power BI）
- 报表需求相对标准

## 五、混合策略详解

### 5.1 推荐的系统组合

```
┌─────────────────────────────────────────────────────┐
│                    自研系统                          │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐               │
│  │   OMS   │ │   WMS   │ │ 库存中台 │               │
│  │ 订单系统│ │ 仓储系统│ │         │               │
│  └─────────┘ └─────────┘ └─────────┘               │
├─────────────────────────────────────────────────────┤
│                    采购系统                          │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐               │
│  │  财务   │ │   HR    │ │   TMS   │               │
│  │ 金蝶/用友│ │ 钉钉    │ │  SaaS   │               │
│  └─────────┘ └─────────┘ └─────────┘               │
├─────────────────────────────────────────────────────┤
│                    可选自研                          │
│  ┌─────────┐ ┌─────────┐                           │
│  │ERP采购  │ │   BI    │                           │
│  │         │ │         │                           │
│  └─────────┘ └─────────┘                           │
└─────────────────────────────────────────────────────┘
```

### 5.2 成本效益分析

| 系统 | 自研成本（3年） | 采购成本（3年） | 建议 |
|-----|---------------|---------------|-----|
| OMS | 300万 | 200万+定制100万 | **自研** |
| WMS | 250万 | 150万+定制80万 | **自研** |
| 库存中台 | 100万 | 无合适产品 | **自研** |
| 财务 | 200万 | 50万 | **采购** |
| HR | 50万 | 10万 | **采购** |
| TMS | 100万 | 30万 | **采购** |

**总成本对比**：
- 全部自研：约1000万
- 全部采购：约500万（但OMS/WMS不好用）
- 混合策略：约750万（核心系统好用）

### 5.3 实施顺序

```
第1阶段：OMS自研 + 财务采购
         ↓
第2阶段：WMS自研 + TMS采购
         ↓
第3阶段：库存中台自研 + BI采购/自研
         ↓
第4阶段：持续优化迭代
```

## 六、决策检查清单

### 6.1 自研前的检查

**OMS自研检查**：
- [ ] 是否有3个以上销售渠道？
- [ ] 是否有复杂的订单逻辑（拆单、合单等）？
- [ ] 是否有技术团队（至少3个后端）？
- [ ] 是否有6个月以上的开发时间？

**WMS自研检查**：
- [ ] 是否有自营仓库？
- [ ] 是否有特殊的仓储策略需求？
- [ ] 是否需要与OMS深度集成？
- [ ] 是否有技术团队支持？

### 6.2 采购前的检查

**财务系统采购检查**：
- [ ] 是否了解金蝶/用友的产品线？
- [ ] 是否评估了与OMS/WMS的集成方案？
- [ ] 是否考虑了多币种、多主体需求？

**TMS采购检查**：
- [ ] 是否了解主要物流商的对接方式？
- [ ] 是否评估了SaaS产品的功能？
- [ ] 是否考虑了成本？

## 七、总结

### 7.1 核心结论

| 系统 | 建议 | 原因 |
|-----|-----|-----|
| OMS | **必须自研** | 多渠道、复杂逻辑、核心竞争力 |
| WMS | **建议自研** | 仓储策略定制、与OMS集成 |
| 库存中台 | **建议自研** | 多仓汇总、与OMS/WMS集成 |
| 财务 | **建议采购** | 合规要求、成熟产品 |
| HR | **建议采购** | 非核心、SaaS成熟 |
| TMS | **建议采购** | 物流商对接复杂、SaaS成熟 |

### 7.2 关键原则

1. **核心系统自研**：OMS、WMS是核心，必须自己掌控
2. **边缘系统采购**：财务、HR等非核心，用成熟产品
3. **集成是关键**：自研和采购的系统要能打通
4. **渐进式建设**：先做最痛的，再逐步完善
